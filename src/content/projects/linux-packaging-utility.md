## Overview
Developed a terminal-based Python utility to simplify the process of preparing Linux desktop applications and Debian packages. The project focused on reducing repetitive manual steps in Linux software packaging by guiding the user through a more structured CLI workflow.

This was valuable because it moved beyond simple scripting and into workflow design: figuring out how user input, filesystem structure, naming conventions, and Linux packaging expectations all fit together in a usable tool.

---

## Development Goals

The project was built to strengthen:

- Python CLI tooling
- Linux filesystem and package-structure understanding
- Automation of repetitive setup steps
- Practical developer tooling for Linux distribution tasks

---

## Implementation

Built the utility as a **terminal-first Python application** that accepts user input and uses it to organize files, package metadata, and expected locations for Linux application setup.

- Worked through path, filename, and file-placement challenges using Python and the `os` library
- Applied knowledge of the Linux directory structure to package layout decisions
- Focused on making the packaging flow more approachable and repeatable

The project helped me better understand how small tooling improvements can remove friction from developer workflows, especially on Linux where packaging steps can be tedious and error-prone.

## Key Achievements
- **CLI Tooling:** Built a Python terminal utility to simplify Linux desktop app preparation and Debian package creation.
- **Filesystem Handling:** Solved challenges involving file naming, placement, and path calculation through user input and Python filesystem logic.
- **Linux Packaging Knowledge:** Applied Linux directory-structure conventions to support more accurate package setup and deployment behavior.
- **Workflow Improvement:** Reduced packaging friction by turning a manual process into a more guided repeatable tool.
