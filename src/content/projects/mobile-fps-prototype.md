## Overview
Developed a Unity-based mobile FPS prototype to explore how first-person systems translate onto phone hardware. The project focused on building multiple interconnected gameplay features inside a constrained prototype environment, including object interaction, inventory behavior, enemy AI, and rendering experimentation.

This served as a practical learning project for understanding how a gameplay-heavy concept behaves when moved into a mobile context, where usability, performance, and rendering choices matter more aggressively than on desktop.

---

## Development Focus

The prototype was built to explore:

- FPS interaction on mobile devices
- Inventory and object interaction systems
- Enemy AI and gameplay-state coordination
- Rendering behavior and visual experimentation in Unity

---

## Implementation

Built the project in **Unity** with **C#**, using the prototype to connect several gameplay systems rather than building only a single mechanic.

- Created player interaction workflows for engaging with objects
- Added inventory-related logic and item handling
- Implemented enemy behavior as part of a playable FPS loop
- Explored rendering and presentation behavior alongside gameplay systems

The project was especially useful as a systems-integration exercise because each feature affected the others. It was less about creating a content-rich game and more about understanding how gameplay building blocks fit together in practice.

## Key Achievements
- **Gameplay Systems:** Built a mobile FPS prototype with object interaction, inventory handling, enemy AI, and first-person combat flow.
- **Mobile Exploration:** Used the project to test how gameplay and usability translated to mobile hardware constraints.
- **Rendering Experiments:** Explored Unity rendering behavior and presentation choices as part of the prototype process.
- **Systems Integration:** Strengthened practical understanding of how multiple gameplay systems interact inside a single prototype environment.
